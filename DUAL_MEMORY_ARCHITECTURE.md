# 双记忆系统架构说明

> OpenClaw Memory System Architecture v2.0  
> 更新日期: 2026-03-12

---

## 架构概述

OpenClaw 采用**双记忆系统架构**，分为**数据层**和**工具层**：

```
┌─────────────────────────────────────────────────────────────┐
│                      双记忆系统架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  数据层 (双记忆系统)                  │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  📁 memory/          🔥 self-improving/    📋 .learnings/ │
│  │  (日记层)            (身份层)              (任务层)     │
│  │  原始对话记录        用户偏好习惯          结构化学习   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              工具层 (记忆管理系统 v2.0)              │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  📊 读取分析 → 📈 生成报告 → 📦 归档维护 → 💾 备份   │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                    输出层                            │   │
│  │  📁 reports/    📁 archive/    📁 .backup/         │   │
│  │  每日报告       归档文件        自动备份             │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 数据层详解

### 1. 日记层 (memory/)

**路径**: `~/workspace/memory/YYYY-MM-DD.md`

**用途**:
- 每日完整对话记录
- 技术操作日志
- 系统配置变更
- 重要事件时间线

**特点**:
- 实时追加写入
- 永久保留
- 原始记录，未加工

**示例**:
```markdown
# 2026-03-12 Thursday

## 今日概要
- 日期: 2026-03-12
- 主要事件: 完成记忆管理系统重构

## 关键成果
1. 创建 Python 版记忆管理系统
2. 完成系统测试 (10/10 通过)
3. 上传文档到 Gist
4. 创建 GitHub 仓库
```

---

### 2. 身份层 (self-improving/)

**路径**: `~/workspace/self-improving/`

**三层结构**:

#### 🔥 HOT (热数据)
- **文件**: `memory.md` (≤100行)
- **加载**: 每次会话自动加载
- **内容**: 用户偏好、活跃模式、临时笔记

#### 🧠 WARM (温数据)
- **目录**: `projects/`, `domains/`
- **加载**: 按需加载
- **内容**: 项目级学习、领域知识

#### 🧊 COLD (冷数据)
- **目录**: `archive/`
- **加载**: 显式查询时加载
- **内容**: 归档的旧记忆

**升降级规则**:
- 模式使用3次/7天 → 提升到 HOT
- HOT 30天不用 → 降级到 WARM
- WARM 90天不用 → 归档到 COLD

---

### 3. 任务层 (.learnings/)

**路径**: `~/workspace/.learnings/`

**日志文件**:

| 文件 | 内容 | 格式 |
|------|------|------|
| ERRORS.md | 错误记录 | [ERR-YYYYMMDD-XXX] |
| LEARNINGS.md | 学习记录 | [LRN-YYYYMMDD-XXX] |
| FEATURE_REQUESTS.md | 功能请求 | [FEAT-YYYYMMDD-XXX] |

**使用规则**:
- 用户纠正 → LEARNINGS.md + corrections.md
- 技术错误 → ERRORS.md
- 功能请求 → FEATURE_REQUESTS.md
- 自反思 → reflections.md

## 工具层详解

### 记忆管理系统 v2.0

**位置**: `~/workspace/scripts/openclaw-memory-system/`

**核心组件**:

| 组件 | 功能 | 文件 |
|------|------|------|
| 入口 | Bash命令转发 | `run.sh` |

| 报告 | 生成报告 | `modules/reporter.py` |
| 归档 | 归档维护 | `modules/archiver.py` |
| 健康 | 健康检查 | `modules/health.py` |

## 工作流程

### 每日工作流程

```
00:00 - daily 任务
  ├── 归档昨天记忆
  ├── 整理今天记忆
  └── 继承昨天待办

02:00 - maintenance 任务 (条件执行)
  ├── 每3天: 整理长期记忆 → MEMORY.md
  └── 每7天: 归档旧记忆 → archive/

03:00 - backup 任务
  └── 创建每日备份

04:00 - health 任务
  └── 系统健康检查

08:00 - report 任务
  └── 生成晨报 → reports/
```

## 关键设计原则

### 1. 互补而非替代

- 工具层**只读取**数据层，不修改
- 原有双记忆系统继续正常工作
- 工具层增强自动化能力

### 2. 自动化优先

- 定时任务自动执行
- 智能判断执行条件
- 减少人工干预

### 3. 可配置

- YAML/JSON双配置支持
- 阈值可调
- 功能可开关

### 4. 健壮性

- 错误处理完善
- 依赖缺失降级
- 日志记录详细
